# Loyalty Intelligence System

**End-to-end ML system for predicting redemption behavior and generating personalized recommendations for 12M+ loyalty program customers.**

Predicts which customers will redeem points, at which retailer, for how much, and recommends the optimal action, channel, and timing.

---

## What It Does

For each customer, the system produces monthly:

| Output | Description |
|--------|-------------|
| **P(redemption)** | Probability of redeeming in next 12 months |
| **P(retailer)** | Most likely retailer (5 retail brands) |
| **Estimated amount** | Points and local currency expected |
| **Recommendation** | What offer, which channel, what urgency |
| **Priority** | High / Medium / Low / Do not contact (based on causal uplift) |

## Architecture

```
BigQuery (12M customers)
    │
    ├── Phase 1: Data Foundation
    │   ├── Customer exclusions (fraud, employees, outliers)
    │   ├── Stratified sample (500K)
    │   ├── Markov funnel (6 states)
    │   └── Feature engineering (74 features, 27 monthly snapshots)
    │
    ├── Phase 2: Modeling
    │   ├── XGBoost cascade (propensity → retailer → amount → revenue)
    │   ├── KMeans clustering (6 behavioral archetypes)
    │   ├── Causal incrementality (PSM + T-Learner for uplift)
    │   └── Decision engine (5-step recommendation rules)
    │
    ├── Phase 3: Productization
    │   ├── Scoring pipeline (chunks of 1M, monthly)
    │   ├── Streamlit dashboard (6 views)
    │   └── FastAPI (6 endpoints)
    │
    └── Phase 4: Feedback Loop
        ├── A/B testing (10% stratified holdout)
        ├── Drift detection (PSI)
        ├── Retraining triggers
        └── Decision rule optimization
```

## Key Technical Decisions

- **74 features in 11 groups** (RFM, points dynamics, retailer mix, funnel state, seasonality, etc.)
- **Anti-leakage design**: strict temporal windows — features use only data before t0, target uses only data after t0
- **Ternary target**: y=0 (no redemption), y=1 (first-time activation), y=2 (recurring redemption)
- **Causal uplift** via T-Learner, not just propensity — prioritizes customers who *respond to treatment*, not just likely redeemers
- **Global priority quantiles** from training set (not per-chunk) for consistent scoring across batches
- **6-state Markov funnel** with tier-dependent churn thresholds (365d for NORMAL/FAN, 730d for PREMIUM/ELITE)

## Models & Methodology

### Cascade (4 models)

| Step | Model | Target | Input |
|------|-------|--------|-------|
| 1 | XGBoost multiclass | P(y=0), P(y=1), P(y=2) | 74 features |
| 2 | XGBoost multiclass | P(retailer) | Customers with P(redeem) > threshold |
| 3 | XGBoost regression | Redemption points | Customers who redeemed |
| 4 | XGBoost regression | Revenue 12m | All customers |

### Clustering (6 archetypes)

| Archetype | Key Traits |
|-----------|------------|
| Heavy Users | High frequency, high spend, fast earning |
| Cazadores de Canje | High redemption rate, digital, points pressure |
| Exploradores | High retailer entropy (multi-retailer) |
| Dormidos | Long inactivity |
| Digitales | Digital-first redemption |
| En Riesgo | Moderate inactivity + expiring points |

### Incrementality (Causal)

- **PSM** (Propensity Score Matching) for ATE estimation
- **T-Learner** (2 XGBoost) for individual CATE (uplift)
- Priority = quantiles of uplift (not propensity), so we target customers who *respond to intervention*

### Decision Engine (5 steps)

1. **Priority** → uplift quantiles (Q80+ High, Q60-Q80 Medium, <Q60 Low, <=0 Do not contact)
2. **Objective** → from funnel state (activate, accelerate, push, retain, reactivate)
3. **Action** → from cluster (with overrides for churn and new customers)
4. **Channel** → digital preference + propensity thresholds
5. **Timing** → urgency based on churn risk, expiring points, inactivity

|-------|---|---------|

| J. Advanced | 6 | ratio_earn_redeem, burstiness, breakage |
